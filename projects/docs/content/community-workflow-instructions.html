Community Workflow Instruction's

  1. Make sure your git branch is clean(no changes) by typing in the terminal git status
  2. Make sure you have the same code from the "master" branch just to make sure all changes from the master branch are in your local instance of NGC by typing in the terminal git pull
  3. Create a new branch to save your work in by typing in the terminal git checkout -b
  4. When you have made the changes you wanted to make to your project, add your changes to the git staged changes by typing in the terminal git add .
  5. Check that all the changes made to the project are staged by typing in the terminal git status
  6. Commit your changes to the branch by typing in the terminal git commit -m ""
  7. Check if you have commitied the changes correctly by typing in the terminal git status. if it shows up green you are good, if not try re-adding/committing.
  8. Confirm pushing your commit to the branch by typing in the terminal git push
  9. Create the upstream by copying / pasting git push upstream message in terminal, and then entereing your SSH password from Getting Started Section. This is what the message should look like in terminalgit push --set-upstream origin
  10. Click "request to git merge" link in terminal
  11. Set Assigne as yourself, Reviewer as Zeta, check both options below the green "git merge" button(delete and squash)
  12. Check your changes by going over to the Changes section.
  13. Wait for the branch master(Zeta), to merge your branch with master.
  14. Close your sandbox!
